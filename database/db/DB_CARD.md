# ТЕХНИЧЕСКИЙ КАРТОЧКА БАЗЫ ДАННЫХ «CORPLINK»

*   **Название СУБД:** `corplink_db`
*   **Реляционное хранилище:** PostgreSQL 16+ 
*   **NoSQL хранилище:** Redis (кэш статусов)
*   **Нормализация:** Третья нормальная форма (3НФ)

## 1. Спецификация таблиц (10 таблиц)
1. `departments` — отделы компании.
2. `users` — учетные записи (с поддержкой soft delete).
3. `chat_types` — справочник типов чатов.
4. `roles` — справочник ролей в чате.
5. `chats` — созданные чаты.
6. `chat_members` — связующая таблица (содержит суррогатный PK UUID).
7. `messages` — история сообщений.
8. `attachments` — прикрепленные файлы (хранение ссылок на S3-хранилище).
9. `message_reads` — факты прочтения сообщений пользователями.
10. `message_audit_logs` — история изменений сообщений.

## 2. Представления (Views) (ЛР 4)
*   `v_chat_members_details` — сводка участников чатов и их ролей.
*   `v_unread_messages_count` — аналитический расчет непрочитанных сообщений.

## 3. Индексы (ЛР 4)
*   `idx_messages_chat_sent_at` — составной B-Tree индекс на `(chat_id, sent_at DESC)`.
*   `idx_chat_members_chat_user` — индекс для быстрого поиска членства в группах.

## 4. Хранимые процедуры и функции (ЛР 5)
*   `sp_create_group_chat` — транзакционное создание чата и назначение владельца.
*   `sp_get_chat_statistics` — сбор статистики сообщений с `IN/OUT` параметрами .
*   `fn_get_user_chats` — возвращает таблицу чатов пользователя (`RETURNS TABLE`).

## 5. Бизнес-триггеры (7 объектов - ЛР 5)
1. `trg_audit_message_edits` — аудит безопасности: копирует старый текст в логи .
2. `trg_check_sender_in_chat` — валидация "узкого места" (запрет отправки не-участниками).
3. `trg_cascade_soft_delete_user` — каскадное виртуальное удаление переписки сотрудника.
4. `trg_limit_private_chat` — контроль лимита участников в личных чатах.
5. `trg_protect_owner_deletion` — запрет на удаление владельца чата.
6. `trg_auto_read_own_message` — авто-прочтение сообщения автором.
7. `trg_delete_empty_chat` — удаление чата без участников.
