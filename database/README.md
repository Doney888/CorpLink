## 📖 1. Описание предметной области

**CorpLink** — это защищенная система мгновенного обмена сообщениями, предназначенная для работы внутри закрытого корпоративного контура организации. 

База данных проекта спроектирована как цифровой архив управленческих решений компании, защищенный от несанкционированных изменений. Система обеспечивает надежное хранение личных и групповых чатов сотрудников, разграничение прав доступа и ведение автоматического триггерного аудита изменений переписки.

## 🛠️ 2. Технологический стек
*   **Реляционная СУБД:** PostgreSQL 16+ (Основное транзакционное хранилище).
*   **NoSQL СУБД:** Redis (Оперативное In-Memory хранилище сессий и статусов сотрудников «в сети»).
*   **Язык программирования панели управления:** Python 3.10+.
*   **Библиотеки:** Streamlit, Psycopg2, Redis-py, Pandas.

## 🏛️ 3. Ключевые возможности базы данных

База данных CorpLink спроектирована в соответствии с требованиями третьей нормальной формы (3НФ), состоит из 10 физических таблиц и содержит ряд оптимизаций и программируемых объектов:

*   **Суррогатные UUID-ключи:** Использование встроенного генератора UUID защищает базу от коллизий ключей при шардировании и распределенном масштабировании.
*   **Представления (Views) (ЛР 4):** 
    *   `v_chat_members_details` — сводная информация об участниках чатов и их ролях.
    *   `v_unread_messages_count` — аналитический расчет непрочитанных сообщений индивидуально для каждого пользователя.
*   **Индексы (ЛР 4):** Составной B-Tree индекс на сообщениях `(chat_id, sent_at DESC)` для быстрой загрузки истории без выполнения тяжелых операций сортировки в ОЗУ.
*   **Программируемый слой (ЛР 5):**
    *   Хранимая процедура `sp_create_group_chat` для транзакционного атомарного создания чата и привязки его создателя в роли Владельца.
    *   Процедура `sp_get_chat_statistics` с использованием `IN` и `OUT` параметров для сбора статистики сообщений.
    *   Триггер `trg_audit_message_edits` типа `BEFORE UPDATE` для автоматического логирования истории правок переписки в таблицу аудита.
    *   Триггер `trg_check_sender_in_chat` типа `BEFORE INSERT` для валидации "узкого места" (проверка прав отправителя перед добавлением записи).

## 🚀 4. Инструкция по локальному запуску

### 4.1. Развертывание PostgreSQL и накат дампа
1. Создайте базу данных с именем `corplink_db` в вашем pgAdmin.
2. Импортируйте и запустите SQL-дамп из папки репозитория:
   `db/corplink_init_db.sql`
   *Скрипт автоматически создаст 10 таблиц, наполнит их тестовыми данными, настроит индексы, представления, 3 процедуры и 7 триггеров.*

### 4.2. Тестирование СУБД (ЛР 4 и ЛР 5)
Для ручной проверки работы триггеров, процедур и планов выполнения `EXPLAIN` используйте готовые тестовые команды из файла:
`db/test_queries.sql`

### 4.3. Запуск интерактивной панели управления (Streamlit)
Для запуска визуальной демонстрации работы базы данных выполните следующие шаги:

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Запустите приложение:
   ```bash
   streamlit run app.py
   ```
   *Панель автоматически откроется в браузере по адресу `http://localhost:8501`. В ней вы сможете отправлять сообщения, проверять работу триггера аудита правок и отслеживать статусы пользователей.*

### 4.4. Запуск NoSQL (Redis)
Убедитесь, что Redis запущен на стандартном порту `6379` [11]. Имитацию изменения статусов сотрудников можно проверить через консольный клиент `redis-cli` командами:
```bash
SET user:presence:11111111-1111-1111-1111-111111111111 "online" EX 300
GET user:presence:11111111-1111-1111-1111-111111111111
TTL user:presence:11111111-1111-1111-1111-111111111111
```
